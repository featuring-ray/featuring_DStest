---

## 1. Overview

### 기능 목적

피처링 솔루션(B2B)과 피처링 스튜디오(B2C)를 연동하여, 브랜드가 인플루언서에게 인스타그램 DM 자동 답장 가이드를 전달하고 성과를 추적하는 기능이다.

### 핵심 가치

| 역할 | 핵심 가치 |
|------|----------|
| **브랜드 (B2B)** | 인플루언서에게 DM 자동화 가이드를 전달 → 성과 자동 비교/관리 → 리포트 자동화 |
| **인플루언서 (B2C)** | 브랜드 가이드를 간편하게 수락 → 자동화 설정/실행 → 성과 확인 |
| **피처링** | 양질의 실시간 데이터 수집 → B2B↔B2C 협업 기능 확장 기반 → 락인 효과 |

---

## 2. 진입점 (Entry Points)

브랜드 담당자는 두 가지 경로로 반응 자동화에 접근한다.

### 2-1. 전역 관리 (Global Entry)

- **위치:** GNB > `반응 자동화 관리` 메뉴 (신규 추가)
- **용도:** 캠페인과 무관하게 자동화 자산을 독립적으로 생성/수정/보관하는 전역 관리 공간
- **GNB 변경:** `캠페인` 섹션에 `반응 자동화 관리` 항목 추가 (tag: `NEW`)

### 2-2. 캠페인 내 진입 (Contextual Entry)

- **위치:** 캠페인 상세 > `반응 자동화 관리` 탭
- **용도:** 특정 캠페인에 연결된 자동화 그룹의 운영 현황 모니터링 및 성과 확인

## 4. 데이터 모델

### 4-1. 핵심 엔티티 관계

```
┌──────────────┐       N:1 (optional)       ┌──────────────┐
│   Campaign   │◄───────────────────────────│  Automation  │
│              │    campaign_id (nullable)   │    Group     │
└──────────────┘                            └──────┬───────┘
                                                   │ 1:N
                                            ┌──────┴───────┐
                                            │  Influencer  │
                                            │  Assignment  │
                                            └──────┬───────┘
                                                   │ 1:N
                                            ┌──────┴───────┐
                                            │    Guide     │
                                            │  (가이드)     │
                                            └──────────────┘
```

- **Automation Group** (반응 자동화 그룹): 독립 엔티티. 캠페인에 종속되지 않음
- **Campaign**: 자동화 그룹을 활용하는 업무 맥락(Context)

### 4-2. Automation Group

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `id` | string | Y | PK |
| `name` | string (1~20자) | Y | 자동화명 |
| `campaign_id` | string \| null | N | 연결된 캠페인 ID |
| `product_brand_name` | string | N | 상품/브랜드명 |
| `status` | AutomationGroupStatus | Y | 그룹 상태 |
| `manager` | string | N | 담당자 |
| `start_date` | date | N | 시작일 |
| `end_date` | date | N | 종료일 |
| `created_at` | datetime | Y | 생성일 |
| `created_by` | string | Y | 생성자 |

### 4-3. 상태 정의

**AutomationGroupStatus (그룹 상태)**

```typescript
type AutomationGroupStatus = '준비 중' | '진행 중' | '중단' | '종료'
```

**GuideDeliveryStatus (가이드 전달 상태)**

```typescript
type GuideDeliveryStatus =
  | 'draft'        // 임시 저장: 인플루언서에게 전송 전
  | 'pending'      // 대기중: 전송했으나 인플루언서 미응답
  | 'delivered'    // 전달 완료: 인플루언서가 수락
  | 'failed'       // 실패: 전달 실패
```

### 4-4. 데이터 정책

| 정책 | 설명 |
|------|------|
| **독립성 보장** | 자동화 그룹은 캠페인 없이 단독 생성/수정/보관 가능 |
| **연결 정책 (MVP)** | 전역 그룹을 캠페인에 불러올 때 **복제본 생성** → 기존 로그/성과 미포함, 0부터 시작 |
| **캠페인 삭제 시** | `campaign_id`만 `null`로 변경. 자동화 그룹 데이터는 보존 |
| **링크 유효기간** | 가이드 초대 링크는 생성 시점으로부터 **7일** |
| **수락 제한** | 동일 인플루언서는 동일 가이드에 대해 **1회만** 수락 가능 |

## 5. 시스템 아키텍처

### 5-1. 데이터 흐름

```
┌─────────────────────┐                    ┌─────────────────────┐
│   피처링 솔루션 DB    │                    │   피처링 스튜디오 DB   │
│   (Solution)         │                    │   (Studio)           │
│                      │                    │                      │
│                      │── A. 순방향 전달 ──→│                      │
│                      │   (가이드 초안)      │                      │
│                      │                    │                      │
│                      │←── B. 상태 동기화 ──│                      │
│                      │  (Viewed/Accepted)  │                      │
│                      │    실시간 API 호출   │                      │
│                      │                    │                      │
│                      │←── C. 성과 집계 ────│                      │
│                      │   (DM수/클릭수)     │                      │
│                      │  배치 1h 또는 1d    │                      │
└─────────────────────┘                    └─────────────────────┘
```

| 흐름 | 방향 | 트리거 | 방식 |
|------|------|--------|------|
| **A. 순방향** | Solution → Studio | 브랜드가 [가이드 전달] 클릭 | API 핸드오버 |
| **B. 상태 동기화** | Studio → Solution | 인플루언서 액션 (Viewed, Accepted) | 실시간 API 호출 |
| **C. 성과 집계** | Studio → Solution | DM 발송/클릭 로그 누적 | 배치 (1h 또는 1d) |

### 5-2. Dual-Track Delivery (가이드 전달 방식)

| 구분 | Track A: 링크 공유 | Track B: 자동 매칭 | Track C: 알림톡 |
|------|-------------------|-------------------|----------------|
| **타겟** | 미가입자/매칭정보 없는 유저 | 스튜디오 가입 + 계정 연동 완료 유저 | 전화번호를 아는 유저 |
| **전달 방식** | 고유 토큰 포함 URL 복사 → 직접 전달 | DB 조회 후 앱 푸시(Push) | 알림톡 발송 |
| **매핑 시점** | Late-Binding (수락 시점) | Pre-Binding (전송 전 확정) | - |
| **가입 필수** | 가입 전/후 무관 (미가입 시 기능 불가) | 가입 필수 | - |
| **우선순위** | **P0** | **P2** | **P2** |

### 5-3. 단계별 처리 흐름

| 단계 | 주체 | 행동 | 백엔드 처리 |
|------|------|------|------------|
| 1 | 브랜드 | 솔루션에서 DM 가이드 작성 후 전송 클릭 | Solution → Studio API 호출 (초안 데이터 생성) |
| 2 | 인플루언서 | 스튜디오 접속, 도착한 제안 확인 | Studio DB에 '대기 중' 상태로 노출 |
| 3 | 인플루언서 | 내용 확인 후 '시작하기' 클릭 | Studio DB: 자동화 활성화 / Solution에 '수락됨' 신호 전송 |
| 4 | 시스템 | 인스타 DM 수신 → 자동 답장 | Studio DB에 로그 적재 (성공/실패/클릭) |
| 5-1 | 인플루언서 | 스튜디오에서 성과 확인 | Studio에서 통계 시각화 |
| 5-2 | 브랜드 | 솔루션에서 성과 확인 | Studio → Solution 통계 동기화 후 시각화 |

#### US-P0-05: 가이드 링크 복사 & 공유 (Track A)

| 항목 | 내용 |
|------|------|
| **화면** | `button: 링크 복사` |
| **설명** | 사용자는 링크를 통해 인플루언서에게 자동화 가이드를 전달할 수 있다 |

**링크 정책:**

| 정책 | 규칙 |
|------|------|
| 생성 단위 | 가이드(초안) 단위로 1개 링크 생성 |
| 공유 범위 | 동일 링크를 여러 인플루언서에게 공유 가능 |
| 가입 여부 | 링크 공유 시점에 스튜디오 가입 여부 무관 |
| 매핑 시점 | 링크 접속 후 로그인 시점에 인플루언서 매핑 |
| 수락 제한 | 동일 인플루언서 = 동일 가이드 1회만 수락 가능 |
| 재접속 | 이미 수락한 인플루언서 재접속 시 → 기존 자동화 상세로 이동 |
| 유효기간 | 최초 생성 시점부터 **7일** |
| 만료 처리 | 만료 링크 접근 시 수락 불가, 만료 안내 메시지 표시 |

**인플루언서 진입 플로우:**

```
링크 클릭
  ├── 미가입 → 회원가입 → 인스타 연동 → 가이드 화면 랜딩
  └── 가입자 → 로그인 → 가이드 초안 화면 이동
```

| **설명** | 자동화 소속(전역/캠페인)별 다르게 처리한다 |

**표기 규칙:**
- 피처링(솔루션): 캠페인 연결 여부를 태그/뱃지로 표기
- 전역 자동화 가이드 전달 시 → 인플루언서에게 개별 자동화로 인식
- 캠페인 하위 자동화 전달 시 → 인플루언서에게 캠페인 하위 자동화로 인식

- 특정 템플릿 일괄 적용 가능
- 버튼 링크는 인플루언서별 다르게 부여 가능

### P2

#### US-P2-01: 가이드 상세 확인

- 이미 보낸 가이드 클릭 → 해당 건의 상세 내용 및 처리 상태/로그 확인

#### US-P2-02: 가이드 템플릿 관리

- 템플릿 목록 조회 / 생성 / 삭제
- 자주 사용하는 자동화 가이드를 템플릿으로 저장

#### US-P2-03: 가이드 초안 작성/수정/삭제

- 작성 방식: (a) 템플릿 적용, (b) 직접 입력
- 인플루언서가 수정할 수 없게 특정 영역 잠금 처리
- 임시 저장 시 수정/삭제 가능
- 전달 후에는 수정 불가

#### US-P2-04: 알림톡 전송 (Track C)

- 인플루언서 전화번호 입력 → 알림톡으로 가이드 공유

#### US-P2-05: 스튜디오 가입자 자동 매칭 전달 (Track B)

- 스튜디오 가입 + 인스타 연동 완료된 인플루언서에게 자동 매칭 전달
- 전달 전 API로 유효성 재검증 (탈퇴/연동해지/계정변경 방지)
- 유효 시 → studio_id로 가이드 전달, 상태 '수락 대기'
- 무효 시 → 전송 불가 안내

#### US-P2-06: 가이드 재전달/취소

- 전송 실패 시 재전달 가능
- 인플루언서 수락 전에만 취소 가능

#### US-P2-07: 캠페인 컨텍스트에서 그룹 생성

- 캠페인 상세에서 직접 자동화 그룹 생성 (campaign_id 자동 부여)

## 8. B2C 연동 흐름 요약 (피처링 스튜디오)

### 인플루언서 진입 시나리오

```
브랜드가 가이드 링크 전달
  │
  ├── 인플루언서 미가입
  │   └── 회원가입 → 인스타 연동 → 가이드 화면 랜딩
  │
  └── 인플루언서 기가입
      └── 로그인 → 가이드 초안 화면 이동
```

### 주요 화면 요약

| 화면 | 설명 | 우선순위 |
|------|------|---------|
| 서비스 안내 모달 | 회원가입 전/후 서비스 사용 안내 | P0 |
| 브랜드 가이드 수락 CTA | 가이드 사용 요청 확인 (수락/거절/닫기) | P0 |
| 대시보드 자동화 목록 | 내 자동화 + 브랜드 협업 목록 통합/구분 표시 | P0 |
| 브랜드 파트너십 | 브랜드 초안 사용 요청 목록 | P0 |
| 자동화 설정 | 전달받은 자동화 수정/실행 (잠금 영역 수정 불가) | P0 |
| 브랜드 협업 목록 | 보기 구분: 전체/해야할일/실행중/중단/보관함 | P1 |
| 필터/검색 | 자동화명, 브랜드명, 캠페인명 등 | P3 |

### 대시보드 자동화 목록 컬럼

| 컬럼 | 타입 |
|------|------|
| 자동화 제목 | string |
| 구분 | '내 자동화' \| '브랜드 협업' |
| 상태 | '초안' \| '중단됨' \| '실행 중' |
| 수신 인원 | number |
| 클릭 인원 | number |
| 마지막 수정 일시 | datetime |
