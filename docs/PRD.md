## 배경 및 문제점

현재 캠페인 관리 시스템은 플랫폼별로 데이터가 파편화되어 있고, 정해진 절차를 따라야만 하는 경직된 구조를 가지고 있습니다. 이러한 환경은 다양하고 즉각적인 실무 상황에 유연하게 대처하기 어렵게 만들며, 결과적으로 실무자들이 시스템 대신 엑셀과 같은 외부 도구를 활용해 수기로 데이터를 관리하게 되는 주요한 원인이 되고 있습니다.

---

## 목표

- 뉴캠관 틀 먼저 상용 배포 (베타 뱃지 붙여서)
  - 큰 기능 없이 탭별 테이블 위주
    - 캠페인 목록
    - 캠페인 생성/수정/삭제
    - 인플루언서 탭
    - 콘텐츠 탭
    - 인플루언서 & 콘텐츠 대량 추가/삭제
    - 엑셀 다운로드
  - 기존 캠관 offboarding 안내 팝업 상시 노출
    - 향후 기능 업데이트는 없다
    - 새로 출시될 뉴캠관 사용을 추천한다
  - **목적**: CMS 타임라인 관계 없이 CDO가 자동DM, 피처링 스튜디오와 연동 등을 바로 이어서 작업할 수 있게 하기 위함

- CMS는 뉴캠관 틀 상용 배포 이후 High/Med 요구사항 순서대로 빠르게 기능 붙여나가기
  - 붙여나가기 = **주 1회 or 격주 상용 배포**

- High/Med 요구사항 충족(팔/다리 다 붙어있는 뉴캠관) 시 베타 뱃지 제거
  - 베타 뱃지 제거 시점에 캠관 → 뉴캠관 마이그레이션 진행
  - 대메뉴에서 캠관 삭제 진행
  - **예상 시기: 3월말 or 4월초**

## 인플루언서/콘텐츠 테이블 구조

🟥 = 사용자 수정 불가 / 검색 가능 (콘텐츠 본문/제목, 계정명/닉네임, 카테고리, 커스텀 컬럼)

### 고정 Fields
- 계정명
- 인플루언서 카테고리
- 인플루언서 관리 정보 (이메일, 참고사항, 커스텀 관리 정보)

- 2차 활용 기한
- 광고비

### 커스텀 컬럼 Fields
텍스트, 숫자, URL, 체크박스, 선택, 첨부파일, 노트

### 비용 효율
- **CPE**: 단가 / 인게이지먼트 설정 조합
- **CPV**: 단가 / 조회 수

## 세부 기능 정책

### 캠페인 생성
- 캠페인명: 1-50자
- 캠페인 설명: 1-1000자
- 월별 생성 제한: 스탠다드 10개 / 프리미엄 무제한 (내부 100개)
- 브랜드명: 1-50자
- 시작일은 종료일보다 늦을 수 없음
- 캠페인 별 인플루언서 수: 최대 500명
- 캠페인 별 콘텐츠 수: 최대 500개
- 태그: 최대 50개, 디폴트 컬러 Gray, 글자수 2-20자

### 에러 케이스

**공통**
- `url_parse_failed` — 잘못된 링크 (올바른 형식: https://~, www.)
- `campaign_deleted` — 삭제된 캠페인
- `scrap_failed` — 일시적인 오류 (링크 재등록 필요)

**인플루언서**
- `influencer_already_exists` — 이미 추가된 계정
- `influencer_not_found` — 계정 확인 불가
- `influencer_limit_exceeded` — 추가 한도 초과

**콘텐츠**
- `content_already_exists` — 이미 추가된 콘텐츠
- `content_not_found` — 콘텐츠 확인 불가
- `content_limit_exceeded` — 추가 한도 초과 (501개째일 때)

## IB 질답 (2026-01-14)

**리포트 방향**
- 캠페인팀은 향후 뉴캠관의 정형화된 리포트로 통일
- 성과 TOP10 위주로 풍부하게 정형화된 리포트 구성
- 리포트 구성 재료: 캠페인 Summary, 성과 TOP10 콘텐츠, 콘텐츠 지표별 선택 조회, TOP10 콘텐츠 광콘분과 비슷한 분석

**연동 flow**
- 뉴캠관 ↔ 콘.트 연동 시 캠페인팀 use case: 이미 만들어진 콘.트를 뉴캠관에 불러와서 연동
- 1) 뉴캠관 먼저 생성 후 파생 콘.트 생성 / 2) 이미 생성된 콘.트를 뉴캠관과 연동 → 둘 다 지원해야 함

| No | 질문 | 답변 |
